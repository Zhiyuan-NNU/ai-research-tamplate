# 唤醒.txt 标准模板

【环境准备】  
（如有需要，在此填写工作目录）  
cd 项目根目录  
$env:HTTPS_PROXY="http://127.0.0.1:7897"  
$env:HTTP_PROXY="http://127.0.0.1:7897"  

【唤醒对象】  
唤醒“{子AI名称}”。  

【文件作用说明】  
这是本项目的 AI 唤醒文件。  
你的任务不是立即开始生成内容，而是先完成初始化，恢复当前工作状态，再进入本轮任务。  

【初始化任务】  
请按顺序读取以下文件：  

1. CLAUDE.md  
2. AI指令\{对应工作指令文件名}  
3. AI记忆文件\{对应记忆文件名}  
4. {补充参考文件1}  
5. {补充参考文件2}  

如当前任务涉及具体代码模块，请继续读取对应代码文件夹下的说明.md。  

【工作要求】  
1. 先完成阅读，不要跳过初始化直接开始输出。  
2. 优先理解项目结构、当前角色和任务边界。  
3. 默认保持当前项目结构、术语和文件组织方式不变。  
4. 不要擅自扩大任务范围。  
5. 不要把推测内容当作既定事实。  
6. 个性化要求、具体研究内容和临时任务，以后续对话补充为准。  

【初始化完成后的反馈格式】  
读取完成后，请先反馈以下内容：  

1. 你当前的角色是什么；  
2. 你已恢复的关键信息有哪些；  
3. 你理解的当前重点任务是什么；  
4. 如有不明确之处，请明确指出。  

在完成上述反馈后，再等待进一步指令或继续执行当前任务。
