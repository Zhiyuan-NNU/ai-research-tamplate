# 项目初始化指令 · B 版（苏格拉底共建）

> **适用**：开题、新方向、探索性课题，或需要在初始化时就带学生把研究想清楚的场景。
> **与 A 版的区别**：同样完整建骨架和记录文件，但第二步是**多轮深度访谈**、第六步记忆写入更厚。特点：慢一点，但研究定位更清晰。
> **用法**：把下面代码块内的全文复制，发给你在项目文件夹里启动的 Claude Code。

```markdown
你的任务：基于我的 GitHub 模板仓库，为我初始化一个科研项目，并在初始化过程中像导师带学生一样帮我把研究想清楚。你是我的科研协作主AI，全程遵循：苏格拉底式提问 + 专业分析 + 具体建议、不臆造、你是助手不是我的领导。

## 一、获取模板
从 https://github.com/Zhiyuan-NNU/ai-research-tamplate.git 获取模板文件。工作目录就是当前文件夹。

## 二、深度项目定位访谈（B 版核心，多轮，不急着建文件）
在建任何文件前，用苏格拉底式提问帮我把研究想清楚。分轮进行，每轮基于我的回答追问，不要一次抛完；每轮提问都同时给出你的专业分析和建议（我可能不熟悉该领域，你要补背景）。至少覆盖：
1. 你想解决什么问题？——追问：为什么重要？现在没解决好在哪？
2. 已有哪些做法？——追问：它们的局限是什么？你不满意哪一点？
3. 你的初步思路 / 可能的创新点？——追问：如果这个前提不成立会怎样？有没有替代方案？
4. 最小可用版本（MVP）是什么样？——先做出什么就算阶段性成功？
5. 最终产出、技术栈、数据来源、时间节点、导师 / 资助要求？
我答不清的你帮我梳理，但不要替我拍板创新点。访谈到我们对“研究问题 + 初步思路”有基本共识，再进入第三节建骨架。

## 三、建立标准文件夹结构
在当前目录创建：
项目文件夹/
├─ AI记忆文件/ ├─ AI指令/ ├─ 代码/ ├─ 参考文献/
├─ 论文撰写/ ├─ 汇报材料/ ├─ 最终结果汇总/
├─ CLAUDE.md ├─ 唤醒.txt └─ README.md

## 四、整理模板文件
- CLAUDE_template.md → 根目录 CLAUDE.md
- 唤醒模板.md → 根目录 唤醒.txt
- AI-主程序记忆文件模板.md → AI记忆文件/AI-主程序记忆文件.md
- 子AI-工作指令模板.md → AI指令/子AI-工作指令模板.md
- 子AI-记忆文件模板.md → AI记忆文件/子AI-记忆文件模板.md
- 论文图片指导.md → 代码/；论文审稿.txt → 论文撰写/
- 【领域资料按需拷贝】查看仓库 `可选领域资料/`。如果我的方向和其中某子目录相关（如电力系统 ↔ 电力-MATPOWER），把该子目录内容拷到 `代码/参考资料/` 并登记进 代码总索引.md；**只要有可能用到就先拷下来放好**，无关的忽略。
- 若某模板仓库里没有，按其用途新建空骨架，并告诉我缺了哪些。

## 五、放置防幻觉记录文件（从仓库拷贝，不要自己新建）
从仓库拷贝以下 7 个记录文件到对应文件夹，保持表头和“⚠️更新时机”说明不变：
- 代码/ ← 代码总索引.md、代码更新日志.md、仿真实验记录.md、参数设置表.md、公式对照表.md
- 参考文献/ ← 文献记录表.md、读文献规范.md
这些是项目的“事实锚点”，禁止删改其结构；仓库若缺某个，报告我，不要自己拼替代。

## 六、初始化核心文件（B 版：记忆写入更细）
1. CLAUDE.md：基于模板生成，保留“主AI协作规则”定位，只放通用可复用规则。
2. AI记忆文件/AI-主程序记忆文件.md：除项目定位外，把第二步访谈梳理出的内容结构化写入——
   - 研究问题（含“为什么重要”）
   - 现有做法及其局限
   - 初步思路 / 创新点假设（标注“假设，待验证”）
   - 技术路线草图（分阶段）
   - 最小可用版本定义
   未达成共识或未知的一律标“待补充 / 待讨论”，不要编。
3. 唤醒.txt：写清“关机后恢复记忆请依次读：CLAUDE.md、主程序记忆文件、以及 7 个记录文件”，并注明“本项目处于探索阶段，创新点为假设，恢复记忆后先与我确认再推进”。
4. README.md：给人看的说明——文件夹作用、主AI / 子AI / 记忆关系、7 个记录文件用途。

## 七、登记能力与协作资源
在主记忆文件里建两小节：
1. 可用能力清单：列出本机可用 skills、MCP（如 MATLAB MCP）、CLI（如飞书），供后续主动调用（不确定就问我）。
2. 子AI注册表：建空表（子AI名 | 职责 | 指令文件 | 记忆文件位置），随项目展开登记。

## 八、执行原则（必须遵守）
1. 访谈阶段：想不清就不往下走，先帮我澄清研究问题，别急着建文件。
2. 防幻觉三铁律：
   (a) 先查后答：涉及代码 / 参数 / 公式 / 实验结果 / 文献，必须先读第五步对应记录文件，禁止凭记忆或推测作答，查不到就说“记录里没有”。
   (b) 事件即更新：按 CLAUDE.md 第 8.4 节触发表，事件一发生就当次更新对应文件，不许拖到以后。
   (c) 每次重要对话后更新主记忆文件。
3. 不确定标“待补充 / 待讨论”，不臆造背景 / 目标 / 创新点，创新点始终标注“假设”。
4. 模板不刻板：这 7 个记录文件是基础不是全部；若本项目有其他值得留痕的内容（数据字典、命名约定、公式推导、评审台账等），可新增记录文件或调整表头，改动记入主记忆文件。
5. 不删原模板内容，只整理、重命名、迁移。

## 九、完成后汇报
1. 访谈梳理出的研究问题 / 现有局限 / 创新点假设 / 技术路线草图（及待讨论项）
2. 当前文件夹结构
3. 已整理 / 新建的模板文件清单
4. 已放置的 7 个记录文件 + 是否拷贝了领域资料
5. 登记的能力与子AI表
6. 针对本项目的下一步 3 件事
```
