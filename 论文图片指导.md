> **目的**：AI阅读后可直接遵循的绘图规范。以下规则优先级高于默认行为。

> **强制规则（所有图表必须遵守）**：

## 补充一、配色方案

### 1.1 核心原则

| 原则 | 说明 |
|------|------|
| **色盲友好** | 全球约8%男性为红绿色盲，避免仅靠红/绿区分数据 |
| **灰度可辨** | 黑白打印时各系列仍可区分（靠明度差异，而非仅色相） |
| **系列≤7色** | 超过7种颜色人眼难以区分，多于7组数据应使用形状/线型辅助区分 |
| **语义一致** | 同一论文中，同一物理量始终使用同一颜色（如：SOC永远蓝色，功率永远红色） |

### 1.2 推荐学术配色板

#### A. 经典学术配色（6色，覆盖大部分需求）

| 序号 | 名称 | HEX | RGB | 典型用途 |
|------|------|-----|-----|----------|
| 1 | 深蓝 | `#2E5090` | (46,80,144) | 基准/参考方案、SOC曲线 |
| 2 | 砖红 | `#C0392B` | (192,57,43) | 关键对比方案、功率/成本 |
| 3 | 森绿 | `#27AE60` | (39,174,96) | 可再生能源、正面结果 |
| 4 | 琥珀 | `#F39C12` | (243,156,18) | 警告/中间值、光伏出力 |
| 5 | 紫罗兰 | `#8E44AD` | (142,68,173) | 第五方案、储能 |
| 6 | 青灰 | `#5D6D7E` | (93,109,126) | 次要数据、网格/参考线 |

#### B. Nature/Science风格（高对比、期刊偏好）

```
['#E64B35', '#4DBBD5', '#00A087', '#3C5488', '#F39B7F', '#8491B4', '#91D1C2', '#DC9900']
```

- 来源：nrc (Nature Review Colors) 色板
- 特点：饱和度适中、互相区分度高、灰度下仍可辨

#### C. IEEE PES / 工程期刊风格（偏保守沉稳）

```
['#1F77B4', '#FF7F0E', '#2CA02C', '#D62728', '#9467BD', '#8C564B', '#E377C2', '#7F7F7F']
```

- 来源：Matplotlib default (tab10前8色)
- 特点：工程领域最广泛接受，审稿人熟悉

#### D. 色盲安全配色（Wong 2011, Nature Methods推荐）

```
['#000000', '#E69F00', '#56B4E9', '#009E73', '#F0E442', '#0072B2', '#D55E00', '#CC79A7']
```

- **强烈推荐用于正式投稿**
- 8色覆盖三种常见色盲类型（红绿/蓝黄/全色盲）
- 出处：Wong, B. "Points of view: Color blindness." Nature Methods 8, 441 (2011)

### 1.3 色板类型选择

| 数据性质 | 色板类型 | 示例 | 用途举例 |
|----------|----------|------|----------|
| 分类（无序） | Qualitative | 上述A/B/C/D | 不同电厂、不同方案对比 |
| 有序/连续 | Sequential | 单色渐变（浅蓝→深蓝） | 热力图、负荷等级 |
| 双向/偏离 | Diverging | 蓝→白→红 | 偏差图、正负功率、温度变化 |

### 1.4 配色禁忌

| 禁忌 | 原因 | 替代方案 |
|------|------|----------|
| 红配绿 | 红绿色盲无法区分 | 蓝配橙、蓝配红 |
| 彩虹色（jet/rainbow） | 感知不均匀，灰度下完全无法辨认 | viridis/plasma/inferno渐变 |
| 纯黄色线条/细线 | 白底上几乎不可见 | 深黄/琥珀色，或加粗 |
| 半透明叠加超过3层 | 颜色混合后不可预测 | 使用线型/标记区分 |
| 荧光色/过高饱和度 | 刺眼、不专业 | 降低饱和度10-20% |

## 补充三、坐标轴与标注规范

### 3.1 轴标签格式

**标准格式**：`物理量名称/符号 (单位)` 或 `物理量名称/符号 [单位]`

| 正确示例 | 错误示例 | 说明 |
|----------|----------|------|
| `Power (MW)` | `Power/MW` | 用括号或方括号包裹单位 |
| `Time (h)` | `Time(h)` | 括号前有空格 |
| `*P*_net (MW)` | `Pnet(MW)` | 变量名斜体，下标正体 |
| `SOC (%)` | `SOC%` | 百分号也用括号包裹 |
| `CO₂ emission (t/h)` | `CO2 emission t/h` | 下标用下标格式 |
| `Coal consumption (t/h)` | `coal (t/h)` | 首字母大写 |

### 3.2 刻度值格式

| 场景 | 格式 | 示例 |
|------|------|------|
| 常规数值 | 保留1-2位小数，统一位数 | 0.0, 0.5, 1.0, 1.5 |
| 大数值 | 使用千位分隔符或科学计数法 | 1,000 或 1×10³ |
| 百分比 | 末尾加% | 0%, 25%, 50%, 75%, 100% |
| 时间轴 | 使用整数小时或HH:MM | 0, 6, 12, 18, 24 |
| 金额 | 统一单位前缀 | 万元、k¥ |

### 3.3 误差棒与置信区间

| 表示方式 | 适用场景 | 绘制规则 |
|----------|----------|----------|
| 误差棒（error bar） | 多次仿真结果 | 端帽宽度=标记宽度，线宽=数据线0.5-1倍 |
| 阴影区域（shaded band） | 连续数据的置信区间 | 透明度20-30%，用同系列颜色 |
| 箱线图 | 分布统计 | 标注中位数、四分位、异常值 |

> **规则**：图例或caption中必须说明误差棒含义（标准差SD、标准误SE、95%CI等）。

### 3.4 断轴处理

当数据存在大跨度间隙时（如成本从10到10000）：

| 方法 | 适用场景 | 注意事项 |
|------|----------|----------|
| 波浪线断轴 | Y轴非零起点或跳过空白段 | 断轴处画波浪标记，明确标注 |
| 对数坐标 | 数据跨越多个数量级 | 轴标签标注"log scale" |
| 双Y轴 | 两组量纲不同的数据 | 左右轴颜色与对应数据系列一致 |

> **禁忌**：不加断轴标记直接截断Y轴（会夸大差异，误导读者）。

### 3.5 标注（Annotation）规则

| 标注类型 | 使用场景 | 格式要求 |
|----------|----------|----------|
| 箭头+文字 | 指出关键拐点、峰值 | 箭头细线（0.5pt），文字8-9pt |
| 阴影区域 | 标注特定时段（如峰时电价区间） | 透明度10-20%，颜色与数据区分 |
| 水平/垂直参考线 | 阈值线（如SOC上下限） | 虚线，灰色或对应颜色，标注数值 |
| 数据标签 | 柱状图顶部数值 | 仅标注关键值，不要每个点都标 |

```
│   ├─ 多组分布对比 → 多组箱线图
│   └─ 二维密度 → 热力图
│
├─ 流向/关系网络
│   ├─ 能量/资金流向 → 桑基图
│   ├─ 节点间双向关系 → 和弦图
│   └─ 网络拓扑 → 关系图（graph）
│
├─ 层级/构成
│   ├─ 树状层级 → 树图
│   ├─ 嵌套占比 → 旭日图 / 矩形树图
│   └─ 漏斗流程 → 漏斗图
│
└─ 多维评估
    ├─ 多方案多指标 → 雷达图（≤8维）
    └─ 高维数据探索 → 平行坐标图
```

## 补充五、常见错误与反面案例

> AI绘图时**必须避免**以下错误。

### 5.1 严重错误（会被审稿人直接指出）

| 错误 | 问题 | 正确做法 |
|------|------|----------|
| **在图片上方加标题** | 论文图表标题应在Figure caption中，不在图上 | 删除title，仅保留轴标签和图例 |
| **使用子图/subplot** | 多子图布局应由LaTeX/Word排版控制 | 每张图独立输出为单独文件 |
| **3D饼图/3D柱状图** | 透视变形导致面积/高度感知失真 | 使用2D饼图或2D柱状图 |
| **Y轴不从0开始（柱状图）** | 柱状图面积代表量值，截断底部夸大差异 | 柱状图Y轴必须从0开始 |
| **无单位的轴标签** | 审稿人无法判断量纲 | 所有轴标签加单位 |
| **分辨率不足** | 打印模糊 | ≥300dpi位图或矢量格式 |
| **使用JPG格式** | 有损压缩产生块状伪影 | 用PNG（位图）或SVG/PDF（矢量） |

### 5.2 中等错误（影响论文质量）

| 错误 | 问题 | 正确做法 |
|------|------|----------|
| **图例遮挡数据** | 关键数据被图例覆盖 | 移动图例到空白区域 |
| **线条颜色太浅/太细** | 打印或缩放后不可见 | 线宽≥1.0pt，避免浅色细线 |
| **刻度过密** | 数字重叠难以阅读 | 减少刻度数量，5-7个主刻度为佳 |
| **双Y轴刻度不对齐** | 零点或网格线不一致造成误导 | 手动对齐零点和网格 |
| **颜色含义不一致** | 图1中蓝色=方案A，图2中蓝色=方案B | 全文同一物理量/方案使用同一颜色 |
| **图例顺序与视觉顺序不一致** | 堆叠图中图例顺序与堆叠顺序相反 | 图例顺序与数据视觉顺序一致 |
| **网格线太粗/太深** | 喧宾夺主 | 网格线浅灰（#CCCCCC）、0.3pt |
| **过多数据系列（>7条线）** | 无法区分 | 拆分为多张图，或使用热力图 |

### 5.3 细节问题（体现专业度）

| 问题 | 改进 |
|------|------|
| 坐标轴标签首字母未大写 | `Power (MW)` 而非 `power (MW)` |
| 变量名未用斜体 | *P*_net 而非 Pnet |
| 图例用默认名（Series1） | 改为有意义的名称（"POFC output"） |
| 柱状图无间距 | 柱间距设为柱宽的20-50% |
| 时间轴标签倾斜 | 优先调整刻度间隔，避免倾斜标签 |
| 小数位不统一 | 同一轴所有刻度保持相同小数位 |

### 6.3 各类图表的期刊风格模板要素

#### 调度结果图（堆叠面积/柱 + 电价折线）

```
必备要素:
- X轴: Time (h), 0-24整数刻度
- Y轴(左): Power (MW), 从负值（充电）到正值（发电）
- Y轴(右): Electricity price (¥/MWh), 如使用TOU电价叠加
- 堆叠顺序(从下到上): 煤电 → 核电 → 风电 → 光伏 → 储能放电
- 负值区域: 储能充电、抽水蓄能
- 叠加折线: 净负荷曲线（虚线）
- 图例: 水平排列在图的上方
- 配色: 每种电源固定颜色（全文一致）
```

#### SOC曲线图

```
必备要素:
- X轴: Time (h), 0-24
- Y轴: SOC (%), 0-100%
- 参考线: SOC_max 和 SOC_min (水平虚线, 灰色)
- 起止点: SOC(0) = SOC(24) 标注（日循环约束）
- 如有多台储能: 不同线型+颜色区分
```

#### 成本对比图（分组/堆叠柱状图）

```
必备要素:
- X轴: 方案名称（Case 1, Case 2, ...）
- Y轴: Cost (×10⁴ ¥) 或 Cost (k¥)
- 堆叠分量: 购电成本 / 弃电惩罚 / 碳成本 / 储能折旧 等
- 数据标签: 仅在总成本顶部标注总值
- 柱间距: 组间距 > 组内间距
- Y轴从0开始
```

#### 可行运行域图（散点+边界）

```
必备要素:
- X轴: Net power output, P_net (MW)
- Y轴: Hydrogen production, H₂ (kg/h)
- 散点: Aspen仿真工况点（实心圆, 小标记）
- 边界线: 代理模型边界（实线, 2pt）
- 填充: 可行域内部浅色半透明填充
- 标注: 关键边界条件（LF_min, LF_max, MIXED_min, MIXED_max）
- 等值线(可选): LF等值线或MIXED等值线（虚线, 标注数值）
```

#### Pareto前沿图

```
必备要素:
- X轴: 目标1（如 Total cost (×10⁴ ¥)）
- Y轴: 目标2（如 CO₂ emission (t)）
- 散点: 所有非劣解（实心圆）
- 连线: Pareto前沿连线（实线）
- 标注: 极端解和折衷解（箭头+文字）
- 参考点: 理想点/单目标最优解（空心标记）
```

#### 雷达图（多方案综合评价）

```
必备要素:
- 维度: 5-8个评价指标（成本、排放、消纳率、灵活性等）
- 归一化: 所有指标归一化到[0,1]，方向统一（越大越好）
- 填充: 半透明填充（透明度20-30%）
- 方案数: ≤4个（超过则改用表格或平行坐标）
- 标注: 每个维度的物理含义和单位在caption中说明
```
